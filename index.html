  1. Introducción

    El presente documento detalla las especificaciones técnicas y la identidad visual para el desarrollo de la plataforma de gestión de la distribuidora de productos artesanales Tierra Viva. Esta plataforma está diseñada para optimizar los procesos administrativos internos, mejorar el control de stock y ventas, y ofrecer a los clientes una experiencia sencilla y eficiente para realizar pedidos y consultas.

    El objetivo es crear un sistema robusto, seguro y fácil de usar que facilite la gestión de usuarios, productos, pedidos, pagos y eventos. Además, contará con una interfaz pública para los clientes y una administración centralizada para el equipo interno.

    Se plantea el desarrollo en tres etapas claras para facilitar la implementación y seguimiento del proyecto:

    • Etapa 1: Modelado y preparación técnica (base de datos, definición de flujos y seguridad).
    • Etapa 2: Desarrollo del sistema de gestión interna (panel administrativo, módulos CRUD, control de roles y permisos).
    • Etapa 3: Desarrollo de la web pública y su integración con el sistema administrativo para la gestión y visualización de pedidos por parte de los clientes.

    Este documento busca ser una guía clara para el equipo de desarrollo, donde se especifican las tareas, módulos y funcionalidades para garantizar que todos los involucrados tengan una visión unificada y concreta de lo que se debe realizar.

  2. Especificación General del Sistema

    El sistema contemplará dos perfiles:

    • Administrador: con acceso total a todos los módulos y funcionalidades.
    • Empleado: con permisos limitados según las tareas asignadas.

    Se implementarán controles de acceso y seguridad para garantizar la privacidad y protección de los datos.

  3. Tareas Iniciales Prioritarias

    Antes del desarrollo, se realizarán las siguientes tareas previas:

    • Modelado de base de datos relacional con las entidades:
      • Usuarios
      • Productos
      • Pedidos
      • Pagos
      • Clientes
      • Eventos
      • Stock
    • Definición de flujos y estados para:
      • Pedidos: pendiente, en preparación, enviado, entregado
      • Pagos: pendiente, parcial, completo
    • Diseño de rutas y seguridad, con control de sesiones, validación y autenticación.
    • Definición clara de roles y permisos.

  4. Módulos de Funcionalidad del Sistema (con CRUD)

    A continuación, se detallan los módulos funcionales principales, con descripción clara de sus operaciones y alcance. Las operaciones CRUD (Crear, Leer, Actualizar y Eliminar) están incluidas donde sean requeridas:

      1. Usuarios y Seguridad

        • Login con validación de credenciales
        • Gestión de roles (administrador / empleado)
        • Control de accesos según permisos

      1. Clientes

        • CRUD de clientes (alta, baja, modificación, consulta)
        • Registro de datos: nombre, contacto, zona, frecuencia de compra
        • Asociación con pedidos y pagos

      1. Productos

        • CRUD de productos (alta, baja, modificación, consulta)
        • Carga de imágenes de productos
        • Categorización de productos
        • Control automático de stock según pedidos

      1. Pedidos

        • CRUD de pedidos (alta, baja, modificación, consulta)
        • Seguimiento de estado: pendiente, en preparación, enviado, entregado
        • Asociación con cliente y productos
        • Historial de pedidos por cliente

      1. Pagos

        • CRUD de pagos (registro, modificación, consulta)
        • Visualización de deuda total y pagos parciales por cliente
        • Exportación a Excel

      1. Agenda Interna

        • CRUD de eventos y fechas clave (entregas, cobros, recordatorios)
        • Notificaciones internas
        • Vista tipo calendario

      1. Panel de Administración

        • Dashboard con métricas clave
        • Acceso y gestión a todos los módulos vía menú
        • Alertas automáticas (pedidos sin entregar, stock bajo, deuda pendiente)

      1. Reportes y Exportaciones

        • Generación de reportes filtrados (fecha, cliente, estado)
        • Exportación de datos a Excel y PDF

      1. Sitio Web Institucional

        • Presentación de la empresa y valores
        • Catálogo de productos con filtros
        • Formulario de contacto

      1. Formulario de Pedido Web

        • CRUD parcial: creación de pedidos desde la web
        • Validación de datos obligatorios
        • Envío y vinculación automática al sistema administrativo

  5. Identidad Visual y Logo

    Descripción del Logo

    El logo de Tierra Viva representa la conexión con la tierra y la artesanía tradicional, mediante una imagen simple que combina elementos naturales y orgánicos. Su forma remite a una hoja estilizada que simboliza el crecimiento, la naturaleza y la sustentabilidad, valores centrales de la distribuidora.

    Colores Corporativos

    • Verde Tierra (
      #4CAF50):       Representa la naturaleza, frescura, y el compromiso con productos artesanales y orgánicos.
    • Marrón Tierra (
      #795548):       Evoca la tierra, la solidez, y la conexión con lo artesanal y genuino.
    • Blanco (
      #FFFFFF) :       Para generar contraste, limpieza visual y claridad en la interfaz.

    Aplicación Visual

    La plataforma mantendrá una línea visual coherente con los colores corporativos, utilizando el verde para elementos activos y botones principales, el marrón para títulos y detalles que requieran un tono cálido y natural, y el blanco para fondos y espacios para garantizar buena legibilidad.

    El diseño buscará una estética simple, elegante y funcional, con tipografías claras y un estilo minimalista que priorice la usabilidad y experiencia del usuario.

  6. Presupuesto Estimado

    El presupuesto se divide en tres etapas para facilitar la organización del proyecto, la asignación de recursos y el seguimiento de avances.

    Etapa Descripción Tiempo Estimado Costo Estimado (USD)
    1 Modelado y Preparación Técnica 1 semana (40 horas) 1,200 - 1,600
    2 Desarrollo del Sistema de Gestión Interna 3 semanas (120 horas) 3,600 - 4,800
    3 Desarrollo de la Web Pública e Integración 2 semanas (80 horas) 2,400 - 3,200
    Total Estimado: 7,200 - 9,600
    Nota:
    • Las tarifas estimadas están basadas en un rango estándar de USD 30 a 40 por hora.
    • Incluye análisis, diseño, desarrollo backend y frontend, integración y pruebas básicas.
    • No incluye diseño gráfico avanzado, hosting ni mantenimiento posterior, que podrían presupuestarse aparte.

  7. Conclusión

    Esta documentación presenta una visión clara, organizada y completa del proyecto para el desarrollo de la plataforma Tierra Viva. Aporta a los desarrolladores la información necesaria para planificar, diseñar y ejecutar el sistema, asegurando que cada funcionalidad esté correctamente entendida y alineada con los objetivos del negocio.

    La división en tres etapas facilita el desarrollo progresivo, permite entregas parciales y un control de calidad efectivo, garantizando que el sistema final sea robusto, seguro y fácil de usar tanto para el equipo interno como para los clientes.
